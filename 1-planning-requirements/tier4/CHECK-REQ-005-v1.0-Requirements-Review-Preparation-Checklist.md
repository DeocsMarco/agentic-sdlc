## Purpose

This checklist ensures the requirements review meeting is properly prepared, all materials are ready, and the review process will be efficient and productive. Use this before distributing the review package to stakeholders.

**Use this checklist when:**
- Preparing for a formal requirements review meeting
- Verifying readiness before distributing review package
- Planning a re-review after significant changes

**Time Required**: 15-20 minutes

---

## Prerequisites

Before starting this checklist, ensure you have:

- ☑ Completed SRS Document (from WI-REQ-003)
- ☑ Completed User Stories (from WI-REQ-002)
- ☑ Feasibility Assessment Report (from WI-REQ-004)
- ☑ Stakeholder contact information

## Checklist Items

### Section 1: Document Readiness

☐ **Item 1.1**: SRS Document is finalized (no pending edits or TBD items)
  - **Expected Outcome**: All sections complete; no placeholder text
  - **Verification**: Search for "TBD", "TODO", "placeholder" — zero results
  - **Notes**: _____________________________________________

☐ **Item 1.2**: User Stories are complete with acceptance criteria
  - **Expected Outcome**: All stories have proper format and 3-7 ACs each
  - **Verification**: CHECK-REQ-002 completed and passed
  - **Notes**: _____________________________________________

☐ **Item 1.3**: Feasibility Assessment Report is complete
  - **Expected Outcome**: Clear verdict with all requirements rated
  - **Verification**: CHECK-REQ-004 completed and passed
  - **Notes**: _____________________________________________

☐ **Item 1.4**: Requirements Traceability Matrix is current
  - **Expected Outcome**: All requirements linked to stories and use cases
  - **Verification**: No orphan items in the matrix
  - **Notes**: _____________________________________________

☐ **Item 1.5**: All documents use consistent versions and formatting
  - **Expected Outcome**: Version numbers, dates, and formatting aligned
  - **Verification**: Spot-check headers and footers across documents
  - **Notes**: _____________________________________________

### Section 2: Review Package Assembly

☐ **Item 2.1**: Review package assembled with all required documents
  - **Expected Outcome**: SRS, stories, feasibility report, traceability matrix, risk register
  - **Verification**: Package contains all listed documents
  - **Notes**: _____________________________________________

☐ **Item 2.2**: Cover note prepared with reviewer guidance
  - **Expected Outcome**: Note explains what's included, what to focus on, and how to provide feedback
  - **Verification**: Cover note includes meeting date, time, and agenda
  - **Notes**: _____________________________________________

☐ **Item 2.3**: Open questions list prepared
  - **Expected Outcome**: Any unresolved items from previous steps documented
  - **Verification**: Each open question has context and proposed resolution options
  - **Notes**: _____________________________________________

### Section 3: Meeting Logistics

☐ **Item 3.1**: Review meeting scheduled with all required attendees
  - **Expected Outcome**: Meeting in all calendars with accepted responses
  - **Verification**: All primary stakeholders confirmed attendance
  - **Notes**: _____________________________________________

☐ **Item 3.2**: Meeting room or virtual meeting link set up
  - **Expected Outcome**: Space booked or link generated with screen sharing capability
  - **Verification**: Test link/room availability before the meeting
  - **Notes**: _____________________________________________

☐ **Item 3.3**: Meeting duration adequate for project scope
  - **Expected Outcome**: 2-4 hours depending on project size
  - **Verification**: Agenda fits within scheduled time
  - **Notes**: _____________________________________________

☐ **Item 3.4**: Note-taker assigned
  - **Expected Outcome**: Designated person to capture feedback during meeting
  - **Verification**: Note-taker confirmed and has FORM-REQ-006 ready
  - **Notes**: _____________________________________________

☐ **Item 3.5**: Recording arranged (if permitted)
  - **Expected Outcome**: Meeting will be recorded for reference
  - **Verification**: Permission obtained from attendees; recording tool tested
  - **Notes**: _____________________________________________

☐ **Item 4.1**: Review package distributed ≥3 business days before meeting

  - **Verification**: Total time blocks fit within meeting duration
