## Purpose

This form guides the creation of well-formed user stories following the INVEST criteria (Independent, Negotiable, Valuable, Estimable, Small, Testable). It ensures user stories have complete acceptance criteria, clear value proposition, and sufficient detail for development team estimation and implementation.

**Completed By**: Project Manager, Business Analyst, or Product Owner
**Approver**: Product Owner or Client Representative (validates value and acceptance criteria)

---

## Instructions

1. Start with prioritized functional requirements (FORM-PM-005-002 output)
2. For each functional requirement, create one or more user stories representing user needs
3. Write user story in standard format: "As a [user role], I want [goal/action] so that [benefit/value]"
4. Define acceptance criteria as testable conditions (minimum 3 per story)
5. Validate each user story against INVEST criteria before finalizing
6. Obtain Product Owner/stakeholder approval on acceptance criteria
7. Add completed user stories to product backlog for estimation and planning

**Important**: Focus on user perspective (not technical implementation). Acceptance criteria should be testable pass/fail conditions.

---

## INVEST Criteria Reference

| Criterion | Meaning | Validation Question |
|-----------|---------|---------------------|
| **I**ndependent | Story can be developed independently | Can this be built without waiting for other stories? |
| **N**egotiable | Details can be discussed and refined | Is this flexible enough for team to decide "how"? |
| **V**aluable | Delivers clear value to user/business | Does this provide tangible benefit to stakeholder? |
| **E**stimable | Team can estimate effort required | Is this clear enough for developers to estimate? |
| **S**mall | Completable in one sprint/iteration | Can this be done in 1-2 weeks? |
| **T**estable | Has clear pass/fail acceptance criteria | Can we test if this is "done"? |

**Quality Check**: Every user story must satisfy all 6 INVEST criteria before being considered "ready" for development.

## Validation Checklist

Before finalizing this user story, verify:

- [ ] User story follows "As a... I want... So that..." format
- [ ] Minimum 3 acceptance criteria defined
- [ ] All acceptance criteria are testable (clear pass/fail)
- [ ] All 6 INVEST criteria validated (all checked)
- [ ] MoSCoW priority assigned and justified
- [ ] Product Owner/stakeholder approval obtained
- [ ] Story added to product backlog

## Common Acceptance Criteria Patterns

**Authentication/Login**:
- Given I am on the login page, When I enter valid credentials, Then I am redirected to dashboard
- Given I enter invalid credentials, When I submit, Then I see error message "Invalid username or password"

**Data Entry/Forms**:
- Given I am on the form, When I leave required field blank, Then I see validation error
- Given I fill all required fields correctly, When I submit, Then data is saved and confirmation message displayed

**Search/Filter**:
- Given I am on search page, When I enter search term and click Search, Then results matching term are displayed
- Given no results match, When I search, Then I see message "No results found"

**Reporting**:
- Given I select date range and click Generate Report, When report completes, Then report displays data for selected dates
- Given I click Export, When export completes, Then CSV file downloads with report data
